# HabitBet

**HabitBet** — социальное мотивационное приложение для формирования привычек и достижения целей  
через публичные обязательства, социальные прогнозы и геймификацию.


## Идея проекта

Пользователь формулирует публичную цель с дедлайном и подтверждает её серьёзность  
автоматической ставкой “за себя”.

Другие пользователи могут делать прогнозы:
- **за** выполнение цели
- **против** выполнения цели

После дедлайна автор загружает доказательство выполнения цели (фото/видео),  
а итог определяется с помощью **арбитража** — голосования случайных пользователей.

Результат влияет на:
- распределение внутренней валюты,
- рейтинг пользователей,
- лидерборд.


## Архитектура проекта

Проект реализуется по архитектуре **клиент–сервер**.

```

habitbet/
├── backend/   # Серверная часть (FastAPI, PostgreSQL)
├── mobile/    # Мобильное приложение (Flutter)
└── README.md

````


## Технологический стек

### Backend
- **Python**
- **FastAPI** — REST API
- **PostgreSQL** — реляционная база данных
- **SQLAlchemy** — ORM
- **Alembic** — миграции БД
- **JWT** — аутентификация
- **Pydantic** — валидация данных и конфигурация

### Client
- **Flutter (Dart)** — кроссплатформенное мобильное приложение (Android / iOS)


## Запуск backend (локально)

### 1. Перейти в папку backend

```bash
cd backend
````

### 2. Создать и активировать виртуальное окружение

```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 3. Установить зависимости

```bash
pip install -r requirements.txt
```

### 4. Создать файл `.env`

```bash
cp .env.example .env
```

При необходимости отредактировать значения в `.env`.

### 5. Запустить сервер

```bash
uvicorn app.main:app --reload
```

### 6. Проверка

* Healthcheck: `http://127.0.0.1:8000/api/v1/health`
* Swagger UI: `http://127.0.0.1:8000/docs`


## Переменные окружения

Основные параметры конфигурации задаются через переменные окружения:

* `DATABASE_URL` — строка подключения к PostgreSQL
* `JWT_SECRET` — секретный ключ для подписи JWT
* `JWT_ALG` — алгоритм JWT
* `DEBUG` — режим отладки
* `MEDIA_DIR` — директория для хранения загруженных файлов

Пример доступен в файле `.env.example`.
